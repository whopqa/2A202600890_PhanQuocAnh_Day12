# Deployment Information

## Public URL

```text
https://YOUR-RENDER-SERVICE.onrender.com
```

Replace this placeholder after the Render service is live.

## Environment Variables Set

Set or verify these in Render:

- `PORT` - injected by Render
- `REDIS_URL` - injected from `ai-agent-redis`
- `AGENT_API_KEY` - generated by Render or manually set
- `JWT_SECRET` - generated by Render
- `LOG_LEVEL=INFO`
- `RATE_LIMIT_PER_MINUTE=10`
- `MONTHLY_BUDGET_USD=10.0`
- `ENVIRONMENT=production`
- `APP_VERSION=1.0.0`

## Test Commands

Set local shell variables first:

```bash
BASE_URL="https://YOUR-RENDER-SERVICE.onrender.com"
API_KEY="YOUR_RENDER_AGENT_API_KEY"
```

### Health Check

```bash
curl "$BASE_URL/health"
# Expected: {"status":"ok", ...}
```

### Readiness Check

```bash
curl "$BASE_URL/ready"
# Expected: {"ready":true,"redis":"ok", ...}
```

### Authentication Required

```bash
curl -X POST "$BASE_URL/ask" \
  -H "Content-Type: application/json" \
  -d '{"user_id":"test","question":"Hello"}'
# Expected: 401
```

### API Test With Authentication

```bash
curl -X POST "$BASE_URL/ask" \
  -H "X-API-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"user_id":"test","question":"Hello"}'
# Expected: 200
```

### Conversation History

```bash
curl "$BASE_URL/history/test" \
  -H "X-API-Key: $API_KEY"
# Expected: Redis-backed messages for user_id=test
```

### Rate Limiting

```bash
for i in {1..15}; do
  curl -X POST "$BASE_URL/ask" \
    -H "X-API-Key: $API_KEY" \
    -H "Content-Type: application/json" \
    -d "{\"user_id\":\"rate-test\",\"question\":\"test $i\"}"
  echo
done
# Expected: eventually returns 429
```

## CI/CD

This repo includes GitHub Actions workflow:

```text
.github/workflows/day12-ci-cd.yml
```

CI runs on every push and pull request to `main`:

- Install Python dependencies for `06-lab-complete`
- Compile Lab 03 Render app
- Compile final project app code
- Run `06-lab-complete/check_production_ready.py`
- Validate `06-lab-complete/docker-compose.yml`

CD can trigger Render automatically after a successful push to `main`.

To enable CD:

1. Open Render dashboard.
2. Go to the deployed web service.
3. Open **Settings**.
4. Copy the **Deploy Hook** URL.
5. In GitHub repo, open **Settings > Secrets and variables > Actions**.
6. Add a repository secret:

```text
RENDER_DEPLOY_HOOK_URL=<your Render deploy hook URL>
```

If the secret is not set, the workflow still passes CI and skips the deploy step.
